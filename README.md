**Звіт

Контейнеризація застосунку за допомогою Docker**

Мета роботи

Метою контейнеризації є створення ізольованого, передбачуваного середовища для запуску застосунку незалежно від операційної системи.
Контейнеризація забезпечує:

портативність застосунку;

швидке та легке розгортання;

ізоляцію залежностей від локального середовища;

зручність тестування та масштабування;

спрощення DevOps-процесів та командної роботи.

Виконані етапи
1. Підготовка проєкту

Було створено простий Python-застосунок (ConsoleFile.py), який запускає HTTP-сервер і повертає текст:

Hello World from Container!


Також створено requirements.txt та Dockerfile.

2. Збірка Docker-образу

У кореневій директорії проєкту виконано команду:

docker build -t consolefile/app:1.0 .


Docker зібрав образ Python + Flask + вихідний код.

3. Запуск контейнера

Контейнер було запущено у фоновому режимі з пробросом порту та змінною середовища:

docker run -d -p 8080:8080 -e APP_NAME=from_container consolefile/app:1.0


Параметри:

-d — запуск у фоновому режимі;

-p 8080:8080 — відкриття порту 8080 контейнера на хості;

-e APP_NAME=from_container — передача змінної середовища.

4. Підключення тому (volume)

Для зберігання логів:

docker run -d -p 8080:8080 -e APP_NAME=from_container \
  -v C:\logs\myapp:/app/logs consolefile/app:1.0


Тепер контейнер може записувати логи в /app/logs, а хост зберігатиме їх у C:\logs\myapp.

5. Перевірка роботи

Після запуску було відкрито:

http://localhost:8080/


Результат: Прикріплено до гугл класу

Hello World from Container!
